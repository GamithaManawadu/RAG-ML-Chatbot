The #1 Trick - Log Transform the Target
Reduces skewness from 1.9 to 0.12 : the single biggest improvement
▼

House prices are heavily right-skewed most houses cost $100K–$200K, but a few mansions cost $500K+. This skew makes linear models perform poorly because they try to minimise squared errors, and the expensive outliers dominate.

Applying log1p(SalePrice) transforms the distribution to near-normal (skewness: 1.9 → 0.12). This is the single biggest accuracy improvement in the entire notebook, and the same technique was applied to 38 other skewed numerical features.

🏫 School Analogy: Imagine grading students where most score 60–80 but a few score 1000. The average gets pulled way up by those outliers. Log transform is like switching to a percentage scale, it compresses the extremes and makes the distribution fair for everyone.
💡 Key Detail: All predictions are made in log space, then converted back to dollars using np.expm1() (the inverse of log1p). This means the model predicts proportional errors, not absolute dollar errors: a 10% error on a $100K house ($10K) is treated the same as a 10% error on a $500K house ($50K).

Missing Values : Context Matters

19 columns had missing values, but most NaN values don't mean "missing data" : they mean "this feature doesn't exist." For example, PoolQC is 99.5% NaN because 99.5% of houses simply don't have a pool. These were filled with "None" (categorical) or 0 (numerical). LotFrontage was filled with the neighbourhood median since nearby houses share similar lot sizes.

Outlier Removal

Two houses with very large living area (>4,000 sq ft) but suspiciously low prices were removed : these are likely data entry errors or unusual sales (e.g., foreclosures) that would confuse the model.

Other Processing Steps

Ordinal encoding converted quality ratings (Ex > Gd > TA > Fa > Po) into numbers (5, 4, 3, 2, 1) preserving their natural order. Skew correction applied log1p to 38 highly skewed numerical features. One-hot encoding expanded the remaining categorical features. The final feature matrix grew from 79 columns to 216 columns.

6 Models Compared : Linear vs Tree-Based
Ridge, Lasso, ElasticNet, GBR, XGBoost, LightGBM
▼

Linear Models

Ridge (L2): penalises large coefficients, keeps all features.
Lasso (L1): drives weak coefficients to exactly 0 : automatic feature selection.
ElasticNet: combines L1+L2.

Tree-Based Models

GBR: Gradient Boosting with huber loss (robust to outliers).
XGBoost: regularised boosting with column subsampling.
LightGBM: fast gradient boosting with leaf-wise growth.

Lasso's Feature Selection

Lasso eliminated 130 out of 216 features (set their coefficients to exactly 0), keeping only 86 active features. This automatic feature selection is one of Lasso's biggest advantages — it tells you which features actually matter and which are noise.

🏫 School Analogy: Imagine asking 6 different real estate agents to estimate a house's price. Each agent has different expertise (one knows the neighbourhood, another knows construction quality). The best estimate isn't from any single agent : it's the weighted average of all 6, giving more weight to the agents with better track records.
📌 Why Ensembles Work: Linear models and tree-based models make different types of errors. Linear models struggle with non-linear interactions; trees struggle with extrapolation. Averaging their predictions cancels out individual weaknesses, producing a more robust prediction than any single model.

Summary
▼
🎯 Takeaway #1 Log Transform Is Essential. Transforming the right-skewed SalePrice (skewness 1.9 → 0.12) was the single biggest improvement. Without it, models try to minimise dollar errors, which are dominated by expensive houses.
🎯 Takeaway #2 Feature Engineering > Model Selection. Creating 11 new features (TotalSF, TotalBath, HouseAge, etc.) improved every model. In tabular ML, the quality of your features matters more than which algorithm you use.
🎯 Takeaway #3 Context-Aware Imputation Matters. Treating NaN as "doesn't exist" (no pool, no garage) instead of "missing data" preserved critical information. Blindly filling with mean/median would have lost this signal.
🎯 Takeaway #4 Linear Models Won. After proper feature engineering and log transforms, Lasso (0.1141) beat XGBoost (0.1187) and LightGBM (0.1201). The data became approximately linear, and linear models handle high-dimensional one-hot features more gracefully.
🎯 Takeaway #5 Lasso = Feature Selection for Free. Lasso eliminated 130 out of 216 features automatically, keeping only the 86 that actually predict price. This is both a regularisation technique and an interpretability tool.
🎯 Takeaway #6 Ensembles Reduce Risk. Combining 6 models' predictions produces more robust estimates than any single model. Different model types (linear vs tree) make different errors, and averaging cancels them out.

📋 Full Pipeline Summary:

EDA + correlation analysis → outlier removal (2 houses) → log1p target transform → combine train+test → context-aware missing value imputation → 11 engineered features → ordinal encoding (quality scales) → log1p on 38 skewed features → one-hot encoding (79 → 216 columns) → 6 models with 5-fold CV → weighted ensemble → Kaggle submission (RMSE 0.12098, Rank #337)
