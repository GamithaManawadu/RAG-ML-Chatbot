🏥 Medical X-Ray Classification

The Big Picture - What's This Project About?

Medical Augmentation, Conservative by Design

Unlike general image augmentation, medical images need conservative transforms to keep anatomy realistic:

AllowedNot Allowed
Rotation ±10° (slight tilt)Large rotation (lungs don't rotate 45°)
Horizontal flip (lungs are roughly symmetric)Vertical flip (anatomy doesn't flip upside down)
Small zoom ±10%, tiny shifts ±5%Extreme zoom/crop (loses diagnostic regions)
Slight brightness ±10%Colour jitter (X-rays are grayscale)

Training was unstable, validation accuracy swung wildly between 95.3% and 27.3% across epochs. The tiny 16-image validation set means each val batch is essentially random. EarlyStopping restored weights from epoch 8 (best val_loss), but the resulting model only achieved 78.5% on the proper test set.

📌 Lesson: Training from scratch on only ~5,200 images is hard. The model has to learn EVERYTHING, edge detection, texture recognition, anatomical structure, AND disease patterns - from a relatively tiny dataset.

Training was much more stable - val_accuracy steadily improved from 87.6% to 95.5%. The pre-trained features from ImageNet (edges, textures, shapes learned from 1.4M natural images) transferred well to X-rays, even though X-rays look very different from cats and cars!

Result: 89.9% - an 11.4 percentage point jump over Model A, with far fewer trainable parameters and faster convergence.

💡 Why ImageNet Features Work for X-Rays: Early convolutional layers detect universal low-level features (edges, gradients, textures) that exist in ALL images - X-rays included. Only the high-level layers are domain-specific. By freezing the base and only training the head, we leverage this universal knowledge.

Val accuracy reached an impressive 98.4% during training, but test accuracy was only 87.7% — lower than the frozen model (89.9%). What happened?

⚠️ Why Fine-Tuning Hurt: With only ~5,200 training images, unfreezing 20 layers gives the model too much freedom — it memorised training-specific patterns instead of learning generalisable features. The 98.4% val accuracy was misleading because the validation set had only 16 images. This is a textbook case of overfitting due to insufficient data.
🏫 School Analogy: Imagine an experienced radiologist who "fine-tunes" their knowledge by only studying 16 practice cases. They'd start over-adapting to those specific 16 images instead of getting better at general diagnosis. That's what happened here — the model needs 10,000+ images for fine-tuning to help.

Medical Metrics — Why Recall Matters More Than Accuracy
Sensitivity 96.7%, threshold tuning, false negative analysis
▼

In medical screening, not all errors are equal:

❌ False Negative (DANGEROUS)

Patient has pneumonia but model says "Normal" → goes home untreated → condition worsens → possible hospitalisation or death

⚠️ False Positive (Inconvenient)

Patient is healthy but model says "Pneumonia" → gets extra tests → costs money and time, but patient is safe

Confidence Analysis

The model's confidence correlates well with correctness: average confidence on correct predictions was 95.1% vs 79.3% on wrong predictions. This means the model "knows when it doesn't know" — a useful property for flagging uncertain cases for human review.

🔧 Technical Challenge Solved: Grad-CAM is straightforward for flat models, but MobileNetV2 was wrapped as a nested model inside the outer Functional model. The Conv_1 layer lived inside the nested MobileNetV2 and wasn't directly accessible from the outer model's computation graph. The fix required building a dual-output model from the base model's internals, then manually passing through the head layers inside the GradientTape — all within a single connected computation graph.
🏫 School Analogy: Grad-CAM is like asking a student to highlight the parts of an X-ray they used to make their diagnosis. If they highlight the lungs — good, they learned radiology. If they highlight the patient's name tag or image borders — bad, they memorised artifacts instead of learning medicine. This is why Grad-CAM is critical for medical AI and a regulatory requirement.

🎯 Takeaway #1 - Transfer Learning Dominates. Frozen MobileNetV2 (89.9%) crushed the from-scratch CNN (78.5%) by 11.4 points. ImageNet features — even from natural images — transfer powerfully to grayscale medical images.
🎯 Takeaway #2 - Fine-Tuning Needs Sufficient Data. Fine-tuning (87.7%) actually performed worse than frozen (89.9%) on this small dataset. With only ~5,200 training images, unfreezing layers led to overfitting. The model hit 98.4% val accuracy but couldn't generalise. This is a valid, honest finding — not a failure.
🎯 Takeaway #3 - Sensitivity > Accuracy for Medical Screening. Model C's 96.7% sensitivity means it catches nearly all pneumonia cases. The 13 missed cases (3.3%) and 64 false alarms represent a reasonable screening tradeoff — better to over-flag than to miss.
🎯 Takeaway #4 - Grad-CAM Provides Essential Explainability. Heatmaps confirmed the model focuses on lung regions, not image artifacts. This is a regulatory requirement for medical AI (FDA) and critical for clinician trust.
🎯 Takeaway #5 - Confidence Calibration Works. The model is less confident when wrong (79.3%) than when right (95.1%), enabling uncertain cases to be flagged for human review.
🎯 Takeaway #6 - Class Weights Handle Imbalance. The 3:1 Pneumonia:Normal imbalance was addressed with computed class weights (Normal=1.94×), preventing the model from simply predicting "Pneumonia" for everything.

⚠️ Limitations & What Would Improve Results:

The tiny 16-image validation set caused unstable training and unreliable early stopping. With a proper validation split (10–15% of training data), Model A would reach ~85%+ and Model C would likely surpass Model B. The dataset is also from a single institution — external validation on other hospitals' X-rays would test real-world generalisability.
